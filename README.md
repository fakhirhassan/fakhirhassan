## Featured Projects

<details>
<summary><b>RAG-Based Legal Document Retrieval System</b> — Agentic retrieval over unstructured legal corpora</summary>

<br/>

A production-oriented retrieval system that lets legal professionals query large, unstructured document collections in natural language and receive precise, grounded answers with source-level traceability.

| Attribute | Detail |
| :--- | :--- |
| **Stack** | Python · LangChain · LangGraph · Pinecone · OpenAI API · Daytona Sandboxes · AWS |
| **Scale** | Multi-thousand-page legal corpora chunked and embedded into a managed vector index |
| **Performance** | Sub-second semantic retrieval with high top-k precision through tuned chunking and metadata filtering |
| **Security** | Isolated, reproducible Daytona sandboxes for safely executing agent-generated code; scoped API credentials |
| **Impact** | Replaced manual document review with natural-language search, materially compressing lawyer research time |
| **Repository** | [github.com/fakhirhassan/RAG_lawyer](https://github.com/fakhirhassan/RAG_lawyer) |

Built the complete data path — ingestion, preprocessing, transformation of unstructured legal text, chunking, and embedding into Pinecone — with explicit data-quality checks at each stage to guarantee retrieval fidelity. Retrieval and document processing are orchestrated as stateful, tool-using agents in LangGraph, allowing multi-step reasoning across documents rather than single-shot lookup. Daytona sandboxes provision isolated environments so agent-generated code can be executed and tested without touching the host system.

</details>

<details>
<summary><b>Tumor Detection from DNA Sequences</b> — ML classification over 1.3M+ genomic sequences</summary>

<br/>

A machine learning system that classifies tumor-associated mutations directly from raw DNA sequence data, built on a purpose-designed genomic ETL pipeline.

| Attribute | Detail |
| :--- | :--- |
| **Stack** | Python · Scikit-learn · Pandas · NumPy |
| **Scale** | 1.3M+ DNA sequences ingested, cleaned, and feature-engineered |
| **Performance** | Reliable pattern recognition in high-dimensional genetic feature space via tuned K-Nearest Neighbors |
| **Security** | Local, offline processing of sensitive genomic data with no third-party transmission |
| **Impact** | Demonstrates a reproducible path from raw sequence data to a deployable classification model |
| **Repository** | [github.com/fakhirhassan/Gene-Mutation](https://github.com/fakhirhassan/Gene-Mutation) |

The engineering challenge was less the model than the data. I designed an ETL pipeline to extract, transform, and load large-scale genetic data into a training-ready representation, with extensive cleaning and feature engineering to make sequence data tractable for distance-based classification. K-Nearest Neighbors was then applied and evaluated for pattern recognition across the resulting high-dimensional space.

</details>

<details>
<summary><b>ANIAD — Text to Animation Pipeline</b> — Generative 2D→3D animation from natural language</summary>

<br/>

An end-to-end generative pipeline that converts a plain-text description into a rendered 3D animation, chaining multiple state-of-the-art vision and generative models.

| Attribute | Detail |
| :--- | :--- |
| **Stack** | Python · Wan-AI/Wan2.2 · TripoSR · MIDAS · Trellis · SDXL · Flask · Next.js |
| **Scale** | Multi-model inference pipeline spanning text-to-image, depth estimation, 3D reconstruction, and animation |
| **Performance** | Staged pipeline with cached intermediate artifacts to avoid redundant model inference between runs |
| **Security** | Sandboxed model execution with strict separation between the Flask inference layer and the Next.js client |
| **Impact** | Collapses a traditionally manual 3D animation workflow into a single natural-language prompt |
| **Repository** | [fakhirhassan.vercel.app/aniad](https://fakhirhassan.vercel.app/aniad) |

ANIAD orchestrates SDXL for generation, MIDAS for monocular depth estimation, TripoSR and Trellis for single-image 3D reconstruction, and Wan2.2 for animation synthesis, all behind a Flask inference service consumed by a Next.js frontend. The core engineering work is the integration layer — normalizing tensor formats, managing GPU memory across sequential model loads, and keeping intermediate representations consistent as data moves between models.

</details>
